## 一、背景与机会

### 1.1 问题

一线知识工作者（PM / 研发 / 运营 / 销售）的日常被三类事务持续消耗：

1. **重复性周期工作**：每日整理 PR、汇总日历、竞品巡检、写周报——内容模式固定，但每天都要人工跑一遍；
2. **会议记录负担**：录音 → 转写 → 提炼纪要 → 拆行动项，一场 1 小时的会议常常要再花 30 分钟做整理；
3. **跨工具碎片化**：信息分散在日历、邮箱、GitHub、Slack、Jira、文档库里，"收集 + 拼装"占据了大量低价值时间。

通用聊天助手（ChatGPT 类产品）解决了"问答"，但没有解决"替我办事"：它们**没有持续记忆**、**不能定时自动执行**、**没有对外部工具的授权管控**——用户既不敢把数据交给它，也无法让它长期值守。

### 1.2 机会与定位

**Cortex Workmate = 随身的 AI 工作搭子**：一个 7×24 常驻、有记忆、能跨工具替你执行重复事务，并在关键动作上征求你授权的个人工作助理。

三个差异化支点：

| 支点 | 含义 | 对应模块 |
|---|---|---|
| **持续协作** | 与 Workmate 的对话是一条连续流，不分 session、有长期记忆 | 对话 / 记忆 |
| **定时自动化** | 用一句话把周期事务托付出去，到点自动执行并回报结果 | 定时任务 / 通知 |
| **可控的工具执行** | 工具调用全透明；动用日历 / 邮箱 / 代码库前先审批，接外部服务前先确认连接 | 审批卡 / MCP 连接 / 通知 |

### 1.3 本期范围（重要）

本期交付为**移动端高保真交互原型**，用于验证产品方向、交互方案与视觉规范，并作为后续正式开发的需求基线：

- 纯前端实现（React + TypeScript），**无真实后端 / 大模型 / OAuth**；AI 回复、工具调用、转写均由脚本化 mock 引擎驱动；
- **定时任务不模拟到点自动触发**：原型演示以「立即运行」代替（运行约 2.6s 后产出结果对话 + 完成通知）；
- 数据持久化在本地（localStorage），提供「重置演示数据」恢复完整演示态；
- 中文为默认语言，界面全量中英双语；浅色 / 深色 / 跟随系统三种主题。

## 二、目标用户与核心场景

**目标用户**：以 Cobo 内部一线知识工作者为原型——产品经理、研发、运营增长、销售 BD。

| 场景 | 用户旅程 | 命中模块 |
|---|---|---|
| 每日 PR 巡检 | "每天早上 9 点帮我整理 GitHub PR" → 到点自动执行 → 通知提醒 → 点开看结果 → 引用结果继续追问 | 对话→定时任务→通知 |
| 产品评审会 | 开会录音 → 选「会议纪要」模板转译 → 看逐字稿与总结 → 发现要发给销售，换「客户总结」模板重新生成总结 | 会议记录 |
| 数据授权 | 让 Workmate 看日历 → 它发起授权申请（用途 / 权限 / 有效期） → 我在对话里或通知中心批准 → 它继续执行 | 审批卡→通知 |
| 接入外部工具 | "把这条同步到 Slack" → Workmate 提示需要连接 Slack → 一键 OAuth 连接 → 继续执行 | MCP 连接卡 |
| 多智能体协作 | 切到 Agents 模式 → 选「合规审查助手」处理 KYT 问题 → 按项目归档会话 | Agents 对话 |

## 三、设计原则

1. **iOS 原生质感**：SF 字体体系、玻璃拟态头部、底部 Sheet / ActionSheet、灵动岛适配的手机框展示；
2. **对话即入口**：定时任务的创建、记忆的保存等核心动作都通过对话完成，功能页只做"管理与回看"；
3. **透明与可控**：每一次工具调用可展开看函数 / 参数 / 结果；敏感能力先审批（含有效期），外部服务先确认连接；
4. **状态稳定**：通知等结构化信息的标题创建即定稿，状态变化只体现在状态药丸，不改标题；
5. **全量双语 + 主题**：界面 chrome 100% 中英双语（默认中文）；浅 / 深 / 跟随系统，切换带全页渐变过渡。

## 四、信息架构

```
登录（Google / 扫码） ──► 新手引导（首次，3 页） ──► 主界面
主界面（底部 3 Tab）
├── 对话
│   ├── Workmate 模式（连续流）── 收藏中心 / 历史搜索 / 通知
│   └── Agents 模式（会话/项目）── 会话历史 / 智能体选择器 / MCP 快捷面板 / 通知
├── 助手
│   ├── 定时任务 ── 任务详情 ── 编辑 / 执行结果对话
│   └── 会议记录 ── 会议详情（转写/总结）── 录音页 / 转译模板选择 / 重新转译
└── 我的
    ├── 个人信息中心
    ├── Workmate 配置：基础配置（人设）/ MCP 服务器（列表/详情）/ 记忆（含回收站）/ 技能
    └── 偏好与支持：语言 / 模式 / 版本 / 重置演示数据 / 退出登录
全局：通知中心（任意页右上铃铛进入）· Toast · 确认弹窗 · 底部 Sheet
```

导航采用「Tab + 覆盖层（overlay）栈」模型：二级页默认从右侧滑入、可逐层返回；底部弹窗（Sheet）用于选择与轻量表单。**方向例外**：录音页自底部弹入（沉浸态）、Agents 会话历史自左侧滑入（抽屉语义）、历史搜索为淡入。

| MEET-05 | 保存 | 「结束」→ 命名弹窗（含「保存后转译」多选框，**默认勾选**；取消则回到暂停态而非丢弃）→ 勾选则保存后**立即开始转译**（MEET-06），不勾选则存为「未转写」；录制中点返回需确认（未保存录音将丢失） |

| MEET-07 | 转译失败与重试 | 转译（上传或转写任一阶段）失败 → 列表药丸变红色「转译失败」外露；详情页显示失败原因 + **「重试转译」**主按钮（重新弹模板选择 → 走统一转译流程），成功后变「已完成」；播放器全程可用，⋯ 菜单（重命名 / 删除）不受影响 |
| MEET-08 | 中断恢复 | 转译进行中刷新 / 杀掉 App → 恢复后该会议标记「转译失败（转译中断，请重试）」，手动重试、不自动续传；**例外**：重新转译（已有内容）被中断 → 回退「已完成」保留原结果。老上传字段数据自动归并为「转译失败」 |

## 七、关键演示动线（验收路径）

1. **登录 → 引导**：扫码登录 → 3 页引导 → 进入 Workmate；
2. **定时任务全链路**：对话输入「每天早上 9 点帮我整理 GitHub PR」→ 任务创建卡 → 助手页看列表 / 详情 → 立即运行 → 通知提醒 → 打开结果 → 「引用提问」追问；
3. **审批链路**：对话请求看日历 → 审批卡第一级直接「同意」（默认授予 7 天有效期）→ 结果续写；或从通知中心点「授权请求 · 日历」跳回卡片审批，回看通知药丸已变「已同意」；
4. **连接链路**：「把这条同步到 Slack」→ 连接卡一键连接 → 通知药丸 待连接 → 已连接；
5. **会议链路**：列表 → 录音（暂停 / 继续 / 结束命名）或上传音频 → 转译选「会议纪要」+ 填总结备注 → 看逐字稿 / 总结（模板 chip + 更新时间）→ ⋯ 重新转译换「行动项清单」（默认仅总结）→ 总结变、逐字稿不变；
6. **失败排查链路**：任务详情打开失败运行 → 「引用提问」让 Workmate 给排查建议；失败会议详情 → 「重新转译」重建；
7. **记忆链路**：对话说「帮我记住：周报每周五下班前发」→ save_memory 工具卡 → 我的 → 记忆列表可见 → 删除入回收站 → 恢复；
8. **收藏与搜索**：收藏一条回复 → 收藏中心跳回原文高亮；历史搜索关键词跳回；
9. **Agents**：切 Agents 模式 → 换「合规审查助手」→ MCP 快捷面板连接 Slack → 会话归档到项目；
10. **偏好**：切换深色（渐变过渡）→ 切英文 → 重置演示数据。
